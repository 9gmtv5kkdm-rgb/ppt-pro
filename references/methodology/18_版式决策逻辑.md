## 📐 第七层：版式决策逻辑（版式模式库）

> 前几层解决了"写什么、怎么讲、怎么配色"，但没解决"这一页到底怎么摆"。
> 本层给出**可执行的版式决策**：拿到一页内容 → 判断意图类型 → 选一个版式模板 → 按模板落位。
> 版式不是审美自由发挥，是"内容意图 → 视线引导"的确定性映射。

### 1. 版式决策树（先判意图，再选版式）

```
拿到一页内容，先问：这一页最重要的任务是什麼？
│
├─ 让人记住一个结论/金句？ ──────────► 金句版式（居中大字）
├─ 让人相信一个数字/数据？ ──────────► 观点+数据版式（左标题/右大数）
├─ 用图表证明一个论点？ ─────────────► 图表证明版式（上标题/下图表）
├─ 把两样东西放一起比？ ─────────────► 对比版式（左右分栏）
├─ 讲先后 / 步骤 / 流程？ ───────────► 流程版式（横向时间轴）
├─ 用两个维度定位（好/坏×重要/不重要）？ ► 矩阵版式（2×2）
├─ 列 3-5 个并列要点？ ─────────────► 列表版式（标题+要点）
├─ 开头定调 / 封面？ ───────────────► 封面版式（标题主导）
├─ 章节切换 / 让人喘口气？ ──────────► 过渡版式（章节名+一句预告）
└─ 收尾复盘 / 呼吁行动？ ───────────► 总结版式（回顾+行动）
```

**决策铁律**：
- 一页只承载**一种**意图类型。意图冲突 → 拆成两页。
- 选版式前先定"视线落点"：观众第一眼必须看到本页的核心（大数字 / 金句 / 图表）。
- 同类型页面在全 PPT 内用**同一版式**，不要一页左图一页右图来回跳。

### 2. 版式模板库（10 种，含落位结构）

> 每个模板给出：适用意图、版面结构（ASCII 示意）、占位元素、关键规则。
> "↑"表示视线优先级（最显眼）。

#### 模板 A：封面版式
```
┌──────────────────────────────────┐
│                                  │
│   [主标题：观点型，≤12字]   ↑    │
│   ────────────                   │
│   [副标题：一句话说明 / 场景]     │
│                                  │
│   演讲人 · 日期        [图/Logo] │
└──────────────────────────────────┘
```
- 占位：主标题（36-60pt 主色）、副标题（18-22pt 深灰）、演讲人/日期（底部小字）、视觉元素（右或底，不抢标题）。
- 规则：封面**不要**放要点；主标题必须是结论或主题，不是"XXX 汇报"。

#### 模板 B：观点+数据版式
```
┌──────────────────────────────────┐
│ [页面标题：观点型]          ↑     │
│                                  │
│ [左：3 个要点支撑，≤15字/条]      │
│                                  │
│               [右：大数字 88%] ↑  │
│               [数字下方一行说明]  │
└──────────────────────────────────┘
```
- 适用：一页要同时讲"为什么"和"有多强"。
- 规则：大数字用主色 + 加粗 + 最大字号（≥44pt），让它从页面跳出来；数字下方用一行小字解释口径。

#### 模板 C：图表证明版式
```
┌──────────────────────────────────┐
│ [页面标题：结论句，不是"趋势图"] ↑│
│                                  │
│      [  图  表  占  50%+  ]  ↑   │
│                                  │
│ [脚注：数据来源 · 时间范围]       │
└──────────────────────────────────┘
```
- 适用：用一张图证明标题。
- 规则：图表标题本身就是结论（见第三层"数据可视化七条铁律"）；图下必标来源与口径；图表选错参照第三层决策树。

#### 模板 D：对比版式（左右分栏）
```
┌──────────────┬──────────────┐
│ [左：方案A] ↑ │ [右：方案B] ↑ │
│ 标题/标签     │ 标题/标签     │
│ · 要点        │ · 要点        │
│ · 要点        │ · 要点        │
└──────────────┴──────────────┘
│ [底：一句对比结论]              │
└──────────────────────────────────┘
```
- 适用：A vs B、现状 vs 目标、我们 vs 竞品。
- 规则：两栏视觉权重对等（同字号、同配色逻辑）；底部一句结论说明"谁更优 / 差异在哪"。

#### 模板 E：流程版式（横向时间轴）
```
[①]──▶[②]──▶[③]──▶[④]──▶[⑤]
 步骤1  步骤2  步骤3  步骤4  步骤5
[一句话说明每步]
```
- 适用：步骤、阶段、生命周期、实施路径。
- 规则：用箭头表示方向；每步只放"名称 + 一句说明"；步骤 >6 时换行或拆页。

#### 模板 F：矩阵版式（2×2）
```
       维度Y高 │ 维度Y低
              │
维度X高 [象限1]│[象限2]
              │
维度X低 [象限3]│[象限4]
              │
       横轴：维度X   纵轴：维度Y
```
- 适用：用两个维度给事物定位（重要/紧急、自研/外采、风险/收益）。
- 规则：四个象限必须标注坐标轴含义；关键项放进对应象限并命名（如"速赢项""待决项"）。

#### 模板 G：列表版式（标题+要点）
```
┌──────────────────────────────────┐
│ [页面标题：观点型]          ↑     │
│  ■ 要点1（动词开头，≤15字）       │
│  ■ 要点2                         │
│  ■ 要点3                         │
│  ■ 要点4（≤5 条，>7 必拆）       │
└──────────────────────────────────┘
```
- 适用：并列的 3-5 个要点、原则、建议。
- 规则：要点互不重叠、都回答"为什么标题成立"；每条前用图标/项目符号统一；避免名词堆砌，动词优先。

#### 模板 H：金句版式（居中大字）
```
┌──────────────────────────────────┐
│                                  │
│     「一句话，最大字号，居中」  ↑ │
│        —— 出处 / 说话人          │
│                                  │
└──────────────────────────────────┘
```
- 适用：开场钩子、章节小结、结尾行动号召。
- 规则：整页只放这一句话；字号全场最大（≥40pt）；可配主色背景或留白，绝不配长段落。

#### 模板 I：过渡版式（章节切换）
```
┌──────────────────────────────────┐
│  0X  [章节名]               ↑     │
│  ────────────                       │
│  [一句预告：这个章节解决什麼]      │
└──────────────────────────────────┘
```
- 适用：大章节之间，也是呼吸页。
- 规则：左上/居中放章节序号 + 名称；配一句"本节能带走什麼"；5 秒内能读完。

#### 模板 J：总结版式（回顾+行动）
```
┌──────────────────────────────────┐
│ [一句话回顾核心结论]        ↑     │
│  · 三句话复盘（不是目录重说）      │
│  · 听完你要做的一件事（行动号召）  │
└──────────────────────────────────┘
```
- 适用：倒数 1-2 页。
- 规则：回顾用"结论重述"而非"目录罗列"；行动号召具体到一步可执行的动作。

### 3. 版式自检清单（落位后逐页过）

- □ 这一页的**意图类型**是否唯一？冲突的拆页了吗？
- □ 视线第一落点是不是本页核心（大数字/金句/图表）？
- □ 标题字号在全 PPT 是否统一？同类型页面版式是否一致？
- □ 文字是否压在页面边缘？左右留白 ≥5%、上下 ≥8% 了吗？
- □ 一页要点是否 ≤5？单条是否 ≤15 字？整页文字是否 ≤60 字？
- □ 图表页：图表占比是否 ≥50%？是否标注来源与口径？
- □ 有无"为了有图而放图"的装饰页？（遮住文字只看图，能懂这页吗？）

### 4. 版式与意图错配速查（典型反例）

| 错配 | 症状 | 修正 |
|------|------|------|
| 数据页用了列表版式 | 关键数字埋在要点里，不跳 | 改"观点+数据"或"图表证明"版式，数字放大 |
| 对比页用了上下堆叠 | 左右难对照，脑子要来回跳 | 改"对比版式"左右分栏 |
| 金句页塞了要点 | 一句话被一堆字淹没 | 改"金句版式"，整页只留那一句 |
| 流程页用了矩阵 | 步骤顺序丢失，像在比高低 | 改"流程版式"横向时间轴 |
| 封面放了 5 个要点 | 开场就信息过载 | 改"封面版式"，只留标题+副标题 |
