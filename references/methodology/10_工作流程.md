## 🔄 工作流程

```
用户请求
  │
  ├─ ① 场景识别
  │     匹配触发词 → 确定S1-S8场景 → 设定风格基调、页数与节奏策略
  │
  ├─ ② 需求澄清（如果关键信息缺失）
  │     谁讲给谁听？什么场合？时长？页数要求？特别禁忌？
  │     是否需要联网搜索补充行业数据？
  │     ⚠️ 只问必要的，不问多余的
  │
  ├─ ③ 结构搭建（应用五层结构法）
  │     ├─ 定中心思想（一句话说清）
  │     ├─ 搭金字塔（一级论点+二级支撑）
  │     ├─ 选故事线（四种故事线选一）
  │     ├─ 画节奏曲线（每页信息密度预估）
  │     └─ 确认页数分配（按场景推荐页数调配）
  │
  ├─ ④ 内容填充（应用内容生成引擎）
  │     ├─ 每页标题提炼（话题→观点）
  │     ├─ 每页要点概括（3-5条，互不重叠）
  │     ├─ 数据可视化建议（图表选择+标注要求）
  │     ├─ 金句埋点（至少3处）
  │     └─ 开场/过渡/结尾设计
  │
  ├─ ⑤ 双轨诊断
  │     ├─ 【快车道】案例库快速筛查 → 秒改常见问题
  │     ├─ 【慢车道】五把刀深度追问
  │     │   ├─ 🔪结构成立吗？
  │     │   ├─ 🔪页面呼吸吗？
  │     │   ├─ 🔪表达精准吗？
  │     │   ├─ 🔪视觉支撑内容吗？
  │     │   └─ 🔪听众能记住什么？
  │     ├─ 【冲突裁决】快慢冲突 → 以慢为准
  │     └─ 【案例更新】新模式反复出现 → 记录到案例库
  │
  ├─ ⑥ 设计建议
  │     ├─ 配色方案（主色+辅助色+文字色）
  │     ├─ 字体方案（标题体+正文体+数字/英文专用体）
  │     ├─ 图表重述（关键页的图表选择与布局建议）
  │     └─ 动画建议（可选）
  │
  └─ ⑦ 输出
        ├─ PPT大纲（每页标题+要点）
        ├─ 每页详细内容提示
        ├─ 设计建议
        └─ 质量自评（五维分数）
```
